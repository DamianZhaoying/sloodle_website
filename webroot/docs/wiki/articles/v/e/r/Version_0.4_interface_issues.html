Controller

  • "sloodle_config" notecard name is case-sensitive but need not be.
  • Error codes are cryptic and give no reference to where they might be translated "ERROR: server responded with status code -514"
  • Not clear what should be done once a config notecard is added - touch object? Move it?
  • Controller must NOT be hidden or any object will give ERROR -514
  • I have multiple notecards all named sloodle_config. Losing track.

Metagloss

  • Chats 2 lines for each request. The second line "Number of definitions found for "xxx": 1" is obvious from the first
  • Why not "Define xxx" instead of the ASCII jargon "/def xxx" or click the box and respond to a query "Chat a word. I'll define it."
  • When you touch it, you should get a web link to the glossary and not the config page - or a choice?

SET Object Dispensor

  • The double menu with 14 options should have the most common objects first. The number buttons are out of order.

  • Object hover text colors are confusing.
    • For Set: Green = need authentication & ready to go
    • For objects: Green = Can't be used. Orange = Ready to go.
