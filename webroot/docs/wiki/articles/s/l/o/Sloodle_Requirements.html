Sloodle Requirements

From SLIS Second Life Wiki

The current stable version of the SLOODLE, version 1.2, runs on Moodle 1.8 or 1.9.

The latest version of SLOODLE, 2.1-beta, runs on Moodle 1.9, 2.0, 2.1, 2.2 or 2.3.

General Requirements

  • A Moodle installation (with Administrator privileges). It helps if you have set up Moodle's Cron job, and has the ability to send email.
  • Access to Moodle files (via FTP or similar)

Network Requirements

  • Your Moodle server needs to be on the public internet, or reachable from your OpenSim grid. It has to allow direct access to files under the "mod/sloodle" directory. Normally this is not a problem, but some Moodle sites have extra security and authentication methods that will get in the way.
  • Everything in SLOODLE 2.x, and some features of SLOODLE 1.x, require the ability to send HTTP messages from your Moodle server to Second Life or your OpenSim grid.
  • For Second Life, this means your web server needs to be able to send outgoing HTTP requests on ports 12046 and 12043. This traffic is sometimes restricted by large Moodle hosts.
  • For OpenSim, this means that you need traffic from Moodle to be able to reach your OpenSim grid. This may involve NAT settings if your Moodle is on the public internet but your OpenSim grid is behind a firewall.

Optional Components

  • The Presenter tool has a feature allowing you to upload PDFs and turn them into presentation slides. This uses ImageMagick, and its PHP bindings.
  • Functionality to post to a blog by sending an email postcard from Second Life or OpenSim is available as a separate module for SLOODLE 1.2, and will probably be included as standard in future SLOODLE versions (starting at 2.2). This requires PHP's IMAP bindings on your Moodle server, and also involves setting up a dedicated email account.
