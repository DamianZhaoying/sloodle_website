SL Release Checklist

From SLIS Second Life Wiki

I'm suggesting following these steps to avoid common problems that we've had before with our in-world releases - Ed, 2010-01-20

Please feel free to add to this or refine it, but try to keep the list of tasks manageable (ideally around 20 minutes, and certainly not more than 1 hour.)

[ ] Do a web setup of your set, so that it knows its site name. If possible, use a different site than the one you developed on.

[ ] Rez each object, then click it, and confirm that the web setup dialog contains the name of your site. If it doesn't, it's either broken or needs resetting and putting back in the rezzer.

[ ] Put the completed set in a distributor, then login as another avatar to confirm that the avatar can use the distributor.

[ ] Rez each object as the alternate avatar, and check the contents of each object to make sure all scripts are full-perms. (TODO: Are there scripts in the toolbar that need to be different?)

[ ] Make sure the distributor itself, and the scripts inside the distributor, can be modified and copied by the Sloodle Developers' Group. (TODO: What are the specific steps?)

Feel free to skip steps if it seems sensible - for example, if you've only altered one object before releasing and otherwise the set is identical to the previous release which has already gone through this process, do the permissions checks on that object and one other object.
